# KET单词闪卡网站 技术设计文档

## 技术选型
### 前端技术栈
- **框架**：纯HTML + CSS + JavaScript（无复杂框架，保证加载速度和兼容性）
- **CSS预处理**：Tailwind CSS v3（快速实现卡通风格UI，响应式设计）
- **图标**：Font Awesome（播放按钮、操作图标等）
- **本地存储**：localStorage（持久化学习进度和单词本数据）
- **语音合成**：Web Speech API（兼容主流浏览器的单词发音功能）
- **PDF导出**：jsPDF + html2canvas（实现单词本导出PDF功能）

### 数据处理
- 先将Word文档中的词汇解析为JSON格式，包含：单词、音标、中文释义、级别（黑1/蓝2/红3）、主题分类（动物/人物/衣服等）、音节划分
- 词汇数据打包为单独的JSON文件，首屏加载完成后异步加载，不影响首屏速度

## UI设计（卡通风格）
### 整体布局
- 顶部导航栏：
  - 左侧：网站标题「KET单词闪卡」
  - 中间：学习模式切换栏（两个按钮：「按级别学习」「按主题学习」），点击可切换学习模式
  - 右侧：实时统计条（总词数 | 已学进度 | 已会：X | 不会：Y），一直可见，数字实时更新
  - 最右侧：单词本按钮、背景切换按钮、导出PDF按钮
- 中间主区域：
  - 默认显示：单词闪卡（占页面70%高度，居中显示）
    - 按级别模式：显示当前级别选择器（黑1/蓝2/红3），可选择要学习的级别
    - 按主题模式：显示主题选择器（动物/人物/衣服等），可选择要学习的主题
  - 点击单词本按钮：切换为单词本视图，顶部有标签页：「全部单词」「已会单词本」「不会单词本」，支持按级别/主题筛选
- 底部操作栏：「不会」按钮、播放按钮、「已会」按钮
- 响应式设计：移动端自适应，按钮大小适合触屏操作

### 单词本视图设计
- 列表形式，每个单词项包含：单词、音标、中文释义、级别标签（黑1/蓝2/红3）
- 已会单词显示绿色对勾标记，不会单词显示红色叉号标记
- 支持分页或滚动加载，单词多的时候不卡顿
- 点击单词可播放发音

### 统计信息设计
- 顶部统计条实时更新：
  - 总词数：所有词汇的总数量
  - 已学进度：百分比形式（如 35%），进度条可视化显示
  - 已会：已标记为会的单词数量
  - 不会：已标记为不会的单词数量
- 点击统计条可展开详细统计面板，显示各级别单词的掌握情况

### 闪卡设计
- 卡片尺寸：PC端 600x400px，移动端 90%屏幕宽度
- 圆角：24px，阴影柔和，卡通感强
- 内容布局：
  - 顶部左侧：级别标签（黑1/蓝2/红3），对应颜色背景（黑/蓝/红）+ 白色文字
  - 顶部右侧：主题标签（如「动物」「人物」），绿色背景 + 白色文字
  - 上部：音标（灰色小字体，居中）
  - 中部：单词（超大号粗体字体，彩色，音节用不同颜色标记）
  - 下部：中文释义（中等字体，深色）
  - 右侧：播放按钮（圆形，橙色，卡通风格）
- 切换动画：卡片从右侧滑入，旧卡片从左侧滑出，过渡时间300ms，缓动效果

### 背景主题
- **森林主题**：绿色渐变背景，卡通树木边框
- **海洋主题**：蓝色渐变背景，卡通海浪边框
- **沙漠主题**：黄色渐变背景，卡通仙人掌边框
- 主题切换：顶部下拉选择，切换时有背景渐变过渡动画

## 核心功能实现方案
### 1. 发音功能
- 使用Web Speech API，兼容Chrome、Edge、360等所有Chromium内核浏览器
-  fallback方案：如果浏览器不支持，隐藏播放按钮，不影响其他功能
- 发音语速：适中，适合单词学习

### 2. 本地持久化
- 存储数据结构：两种学习模式的进度独立存储，互不影响：
  ```json
  {
    // 通用数据
    "mastered": [1, 3, 5...],   // 已会单词ID列表（全局通用，不管哪种模式标记的都会同步）
    "unknown": [2, 4, 6...],    // 不会单词ID列表（全局通用）
    "theme": "forest",          // 当前背景主题
    "lastMode": "level",        // 上次使用的学习模式
    
    // 按级别学习模式进度（独立存储）
    "levelProgress": {
      "黑1": { "currentIndex": 0, "total": 400 },
      "蓝2": { "currentIndex": 0, "total": 400 },
      "红3": { "currentIndex": 0, "total": 400 }
    },
    
    // 按主题学习模式进度（独立存储）
    "topicProgress": {
      "动物": { "currentIndex": 0, "total": 50 },
      "人物": { "currentIndex": 0, "total": 30 },
      "衣服": { "currentIndex": 0, "total": 40 },
      ...
    }
  }
  ```
- 核心逻辑：
  - 单词的「会/不会」标记是全局通用的，不管在哪个模式标记的，所有地方都同步
  - 学习进度是按模式独立存储的：比如你在级别模式学黑1到第100个，切到主题模式学动物到第30个，切回去还能保持各自的进度
  - 页面加载时自动恢复上次使用的模式和对应进度
- 每次操作（标记会/不会、切换模式/级别/主题、切换背景）自动保存到localStorage

### 3. PDF导出功能
- 支持导出「已会单词本」或「不会单词本」
- PDF格式：A4纸，每页10个单词，包含单词、音标、释义
- 导出的PDF带卡通页眉，适合打印

### 4. 性能优化
- 首屏只加载HTML、CSS和核心JS，体积<200KB，加载时间<1s
- 词汇数据（~1MB）异步加载，不阻塞首屏渲染
- 图片资源懒加载，背景图按需加载
- 所有静态资源开启gzip压缩

## 交互流程
```
用户打开网站 → 读取localStorage恢复进度 → 显示当前单词闪卡
↓
用户操作：
├─ 点击播放 → 播放单词发音
├─ 点击「不会」 → 单词加入不会本 → 自动切换下一个单词（滑入动画）
├─ 点击「已会」 → 单词加入已会本 → 自动切换下一个单词（滑入动画）
├─ 点击统计 → 显示统计面板（总词数/进度/已会/不会数量）
├─ 切换背景 → 切换主题背景，过渡动画
└─ 点击导出 → 选择导出类型 → 生成并下载PDF
```
